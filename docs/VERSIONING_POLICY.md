# Versioning Policy

## Standard
This repository follows Semantic Versioning (`MAJOR.MINOR.PATCH`).

## Increment Rules
### MAJOR
Increment when governance changes are backward-incompatible or require mandatory migration.

### MINOR
Increment when adding new standards or materially expanding policy guidance in a backward-compatible way.

### PATCH
Increment for clarifications, typo fixes, formatting updates, and non-substantive corrections.

## Tagging Rules
- Use annotated Git tags in the format `vMAJOR.MINOR.PATCH`
- Tag only after merge to `main`
- Each tag must map to a release note entry describing document-level changes
